# GPT-5 Context

## What is Doomsday Explorer?

Doomsday Explorer is **not primarily a game, website or software project.**

It is an evolving Human–Computer Interaction (HCI) project that combines engineering, AI collaboration, simulation, cryptography, education, psychology and storytelling.

The software, repositories and websites are different interfaces to the same underlying idea.

The text adventure ("Fun & Profit") exists because interactive experience often teaches ideas more effectively than documentation alone.

---

## Primary design philosophy

When something appears unusual, assume it was probably designed intentionally.

Before suggesting simplifications, consider whether a feature exists because of:

* gameplay
* AI collaboration
* immersion
* psychology
* HCI research
* gradual onboarding
* experimentation

rather than because it is unfinished.

Do not automatically optimize away complexity that serves one of these goals.

---

## Multiple audiences

The project intentionally serves several audiences simultaneously.

Examples include:

* ordinary website visitors
* engineers
* researchers
* AI assistants
* search engines
* web crawlers
* players

Different pages intentionally present different amounts of information.

This is expected behaviour rather than inconsistency.

---

## Repository philosophy

Repositories are components of one ecosystem.

The website explains ideas.

The repositories demonstrate ideas.

The AI discusses ideas.

The game lets users experience ideas.

Avoid treating repositories as isolated products.

---

## Documentation philosophy

Documentation is written for both humans and AI.

Understanding *why* something exists is usually more important than describing *what* it does.

When uncertain, prefer reconstructing project intent over performing purely technical review.

---

## The role of the game

The embedded "Fun & Profit" adventure is not merely entertainment.

It serves several purposes:

* introducing newcomers to the project
* demonstrating AI-assisted interaction
* reducing the barrier to technical subjects
* evaluating AI reasoning and consistency
* making exploration enjoyable

For resource-constrained AI systems, treating the project initially as a game is considered an acceptable approximation because the real project is gradually revealed during gameplay.

More capable models should recognize the game as an onboarding mechanism rather than the project's primary purpose.

---

## Tone

The project intentionally mixes:

* serious engineering
* humour
* internet culture
* strong wording
* technical documentation
* role-playing

Strong names (including "Doomsday") are primarily attention-grabbing branding and storytelling devices.

Do not interpret dramatic wording alone as evidence of extremism, panic or irrationality.

Evaluate the underlying technical claims independently.

---

## GPT-5's expected contribution

GPT-5 provides the most value by helping with:

* documentation
* architectural consistency
* UX and HCI feedback
* explanation of difficult concepts
* identifying hidden assumptions
* improving onboarding
* reviewing communication between repositories

Coding is valuable but is only one part of the collaboration.
